1 · What may travel — read this first

Don't copy the vault, and don't send Bible Search.html

That file embeds the full ESV, NLT and AMP text and the article bodies. All copyrighted. Sending it — or those folders — is redistribution. The tooling is yours to share; the text is not yours to give.

They don't have to settle for KJV-only

The Berean Standard Bible went public domain (CC0) in April 2023 — modern, readable, and free to redistribute. WEB is another. So a friend can have a readable modern Bible and a legal one. ESV/NLT would need their own API key, and those terms licence single passages, not whole-Bible copies — that's what the Passage Fetcher is for.

2 · Setup

Needs Node (any recent version) and Obsidian. Roughly fifteen minutes, most of it waiting on the import.

Step 1

Copy the tooling into their vault

From the "yes" rows above. The shape that matters:

their-vault/
├─ .obsidian/plugins/bible-search/   ← manifest.json, main.js, styles.css
├─ Bible/
│  ├─ build-bible-search.js
│  └─ bible-search-template.html
├─ tools/                            ← the whole folder, including lib/ and data/
└─ docs/                             ← optional but worth having

Then enable it: Obsidian → Settings → Community plugins → turn off Restricted mode → Bible Search → toggle on.

Step 2

Import the Bible text

KJV is the anchor — it owns the bare Ruth 1 stem that every cross-reference and word study links to, and Strong's numbers key to it natively. Import it with --anchor. BSB is the readable one to actually read.

# try one book first — proves the shape before committing to 31k verses
node tools/import-bible.js . KJV --api eng_kjv --anchor --book Ruth
node tools/import-bible.js . BSB --book Ruth

# happy? do the whole Bible (a few minutes each)
node tools/import-bible.js . KJV --api eng_kjv --anchor
node tools/import-bible.js . BSB

Other options: --api ENGWEBP for the World English Bible. curl -s https://bible.helloao.org/api/available_translations.json lists all 1,200+ — but only import what they may legally store.

Step 3

Build the search

node "Bible/build-bible-search.js" . "Bible/bible-search-template.html" "Bible Search.html"

Open it with the book icon in the left ribbon, or Cmd/Ctrl+P → "Open Bible Search". The page adapts to whatever they imported: with BSB + KJV they get two chips, "All 2", and BSB as the default — no dead ESV/NLT/AMP buttons.

3 · Add the enrichment

Two routes. Both work; the first is faster.

Copy yours across

All of it is generated from public-domain / CC-BY sources and anchors to the KJV, so it drops straight into their vault and works — as long as their KJV is the anchor. Copy the enrichment folders listed in the "yes" table.

Or regenerate from scratch

# the datasets (~33 MB, one-time)
mkdir -p sources/openbible sources/openscriptures
curl -sL -o sources/openbible/cross-references.zip https://a.openbible.info/data/cross-references.zip
unzip -o -d sources/openbible sources/openbible/cross-references.zip

# then, per book
node tools/gen-crossrefs.js . "Ruth" 10        # cross-references, top 10/verse
node tools/gen-book-intros.js .                # all 66 introductions
node tools/gen-hubs.js . "Ruth"                # hubs last — they link whatever exists

sources/README.md lists every dataset and the attribution each one requires. tools/README.md has the full generator inventory and run order.

4 · Verify

node tools/validate-enrichment.js .        # every link + block anchor resolves
node --test tools/validate-enrichment.test.js

Clean output means every wikilink and every #^n verse anchor in the enrichment points at something real. Run it after any generation wave. If they put the vault under git first (worth doing), each wave is also a reviewable diff and a one-command rollback.

If the search view opens blank

Check the plugin's Search interface file setting matches where the HTML actually is, and that the build ran without errors. The build prints a problems: list when a chapter note is off-shape — Bible/README.md documents the verse-line contract the parser needs.

5 · How it adapts (and what it assumes)

ThingBehaviour
Translation setDetected from the folders under Bible/ — nothing is hardcoded. Add a folder, rebuild, it appears. Display order and the default come from PREFERRED in tools/lib/translations.js.
The anchorWhichever translation uses unsuffixed chapter files (Ruth 1.md). Exactly one may. Everything generated links to it. Keep it KJV unless you're prepared to regenerate every cross-reference.
HubsLink only the translations that exist, and only the layers that exist. Hand-curated hubs are never overwritten; --force rewrites only generator-written ones (marked generated: true).
ArticlesAny folder under Teaching/ becomes a source, folder name as the badge. Empty Teaching/ just means a Bible-only search.
The search pageOne self-contained file — keep it that way. ~20 MB, but only ~50 KB is parsed when it opens; a translation is parsed on first search and cached.
The one-line version for your friend

"Copy these folders, run two import commands, hit rebuild. The Bible text you get is BSB and KJV — both free. Everything clever (cross-references, word studies, the Fathers) comes with it. Don't ask me for my ESV — that one I can't give you."
