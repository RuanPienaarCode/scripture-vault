# Bible — full text and enrichment

Two kinds of folder live here, and they have opposite rules:

| | Folders | Rule |
| --- | --- | --- |
| **Translations** | `ESV/` `NLT/` `AMP/` `KJV/` | Full text. **Read-only to every generator** (PRD TD-1). The set is detected, not hardcoded — add one with `tools/import-bible.js`. KJV is the **anchor**: unsuffixed chapter files, owns the bare `Ruth 1` stem |
| **Enrichment** | `Cross Reference/` `Study Hubs/` `Word Studies/` `Places/` `Catena/` `Commentary/` `Book Intros/` | Generated by `tools/`. Never hand-edit — the next run overwrites you |
| **Reference** | `Reference/` | Hand-curated background notes. No generator; edit freely. Still link-validated |

Enrichment note shapes are frozen in **[[docs/enrichment-layout|docs/enrichment-layout.md]]** —
that spec is authoritative, this file doesn't restate it.

## Translation folders — the shape the search reads

`Bible/build-bible-search.js` walks `Bible/{ESV,NLT,AMP,KJV}/{Book}/` and parses
verses out of the chapter notes. Getting this shape wrong is the #1 reason a
chapter goes missing from search.

### Folder and file names

```
Bible/KJV/Ruth/Ruth 1.md            ← KJV: no suffix
Bible/AMP/Ruth/Ruth 1 (AMP).md      ← everything else: " (TRANS)" suffix
Bible/AMP/Ruth/Ruth (AMP).md        ← book-level note; ignored by the parser (no chapter number)
```

- The book folder name must be the **exact canonical book name** — the builder holds
  all 66 in order (`Genesis` … `Revelation`, and note `Psalms`, `Song of Songs`).
  A folder named `Psalm` or `1Samuel` is reported as missing.
- The filename stem must match its book folder. `Ruth 1.md` inside `Ruth/` ✓;
  `Rth 1.md` is silently skipped.
- **`Bible/KJV/{Book}/{Book} {n}.md` owns the bare `{Book} {n}` stem vault-wide.**
  Nothing else may use it — KJV chapter files are the canonical verse anchors that
  every generated `[[Book n#^v]]` link resolves to (PRD TD-2).

### The verse line

This is the exact contract. One verse per line:

```markdown
**1** Now it came to pass in the days when the judges ruled… ^1
```

| Part | Meaning |
| --- | --- |
| `**1**` | Verse number in bold, at the **start of the line** |
| text | The verse. Collapsed to single spaces when indexed |
| `^1` | Obsidian block anchor at the **end of the line**, matching the verse number |

Regex the builder applies: `/^\*\*(\d+)\*\*\s*(.*?)\s*\^(\d+)\s*$/`

A line missing the bold number, missing the `^n` anchor, or wrapped across two lines
is **not a verse** and won't be indexed or linkable. If a chapter parses zero verses,
the builder says so:

```
AMP: no verses parsed in Ruth 1 (Ruth 1 (AMP).md)
```

### Full chapter note

```markdown
---
tags: [bible, bible/kjv, bible/chapter]
aliases: ["Ruth 1"]
translation: KJV
book: "Ruth"
chapter: 1
---

# Ruth 1

Part of [[Ruth (KJV)|Ruth]] · [[KJV]] · [[Bible]]

**1** Now it came to pass in the days when the judges ruled… ^1
**2** And the name of the man was Elimelech… ^2
```

Frontmatter and the breadcrumb are house style — the search parser only cares about
the folder, the filename, and the verse lines.

## Adding a translation

The four codes are hardcoded in **two** places that must agree:

- `TRANS` in `Bible/build-bible-search.js`
- the filename regex in the same file, which whitelists `(ESV|NLT|AMP|KJV)`

Then add the chip to `#popTrans` in `bible-search-template.html`, populate
`Bible/{CODE}/{Book}/`, and rebuild. Adding a translation is a code change —
unlike adding an article source, which is just a folder.

## Cross references, word studies, places, catena, commentary

Generated — don't hand-write them. See [[tools/README|tools/README.md]] for what to run,
and [[docs/enrichment-layout|docs/enrichment-layout.md]] for the exact note shape,
tag scheme, and the anchor rule.

Source datasets they read from (openbible.info, openscriptures, CCEL) are vendored
and licence-documented in [[sources/README|sources/README.md]]. The vault never fetches at
read time.

## Rebuilding the search

```
node "Bible/build-bible-search.js" . "Bible/bible-search-template.html" "Bible Search.html"
```

`Bible Search.html` is **generated** (~20 MB) — edit
`Bible/bible-search-template.html` instead, then rebuild. The Bible Search tab
reloads itself when the file is rewritten.
